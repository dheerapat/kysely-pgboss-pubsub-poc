Create `Dockerfile` in the project root with the following exact content:

```dockerfile
# syntax=docker/dockerfile:1

# ── Stage 1: install dependencies ────────────────────────────────────────────
FROM oven/bun:1.3.11 AS install
WORKDIR /app

# Install dev deps (for potential type-checking, not copied to runtime)
RUN mkdir -p /temp/dev
COPY package.json bun.lock /temp/dev/
RUN cd /temp/dev && bun install --frozen-lockfile

# Install production deps only (what the runtime stage will use)
RUN mkdir -p /temp/prod
COPY package.json bun.lock /temp/prod/
RUN cd /temp/prod && bun install --frozen-lockfile --production

# ── Stage 2: runtime image ───────────────────────────────────────────────────
FROM oven/bun:1.3.11 AS release
WORKDIR /app

# Copy only production node_modules from install stage (no devDependencies)
COPY --from=install /temp/prod/node_modules ./node_modules

# Copy application source and package manifest
COPY package.json ./
COPY src/ ./src/

# Run as non-root bun user (provided by oven/bun image)
USER bun

# Expose HTTP port (default 3000, overridable via PORT env var)
EXPOSE 3000

# DATABASE_URL must be provided at runtime — no default connection in container
# Example: docker run -e DATABASE_URL=postgres://admin:pass@host:5432/postgres ...
ENTRYPOINT ["bun", "run", "src/index.ts"]
```

Key decisions encoded in this Dockerfile:
- `oven/bun:1.3.11` pinned for both stages (DOCK-01) — not `:latest` to avoid unexpected updates
- `/temp/dev` install is created but NOT copied to runtime (DOCK-03 — no devDeps in final image)
- `/temp/prod` uses `--production` flag → only `dependencies`, no `devDependencies`
- `COPY src/ ./src/` copies the full source tree (Bun runs TypeScript natively, no build step)
- `USER bun` — least-privilege execution using the built-in non-root user from the base image
- `EXPOSE 3000` documents the port; actual binding is via `docker run -p` or Compose

**Critical:** Do NOT add `bun.lock` or `*.lock` to this file. The Dockerfile install stage requires `bun.lock` for `--frozen-lockfile`. If `bun.lock` were excluded, `bun install` would succeed but without reproducible versions.

**Effect on build context:** Docker will not send `node_modules/` (potentially hundreds of MB), `.git/`, or `.env` to the daemon. This reduces build time and prevents secrets from ever entering the image filesystem.
