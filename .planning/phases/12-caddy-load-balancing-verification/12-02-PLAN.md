Run the following verification sequence. Capture and display all output for the human checkpoint in Task 2.

**Step 1: Bring up the full stack (clean start)**
```bash
docker compose down -v 2>/dev/null || true
docker compose up --build -d
```
Wait ~15 seconds for all replicas and Caddy to initialize (pg-boss takes ~2-5s per replica).

**Step 2: Verify Caddy routes to a healthy replica**
```bash
curl -s http://localhost:8080/users
```
Expected: JSON array (may be empty `[]` if no users created yet) with HTTP 200. If non-200, Caddy has not found a healthy upstream.

**Step 3: Observe round-robin routing across replicas**
```bash
# Send 6 POST requests (one per replica)
for i in $(seq 1 6); do
  curl -s -X POST http://localhost:8080/users \
    -H "Content-Type: application/json" \
    -d "{\"email\": \"user${i}@example.com\", \"name\": \"User ${i}\"}" \
    -o /dev/null -w "Request $i: HTTP %{http_code}\n"
done
```
Then check logs for routing spread:
```bash
docker compose logs app --tail=60 | grep -E "POST /users|replica|worker" | head -30
```
Round-robin is confirmed when requests appear in different replica containers (different container IDs in logs).

**Step 4: Verify exactly-once job processing (no duplicates)**
```bash
# Send one POST and watch job processing
curl -s -X POST http://localhost:8080/users \
  -H "Content-Type: application/json" \
  -d '{"email": "singleton@example.com", "name": "Singleton Test"}' \
  -w "\nHTTP: %{http_code}\n"

sleep 3

# Count "user.registered" handler executions across ALL replicas
docker compose logs app --tail=100 | grep -c "user.registered\|handleUserRegistered\|NotificationService\|AuditService"
```
Expected: Each handler fires exactly once per POST — pg-boss advisory lock ensures only one replica processes each job.

**Step 5: Verify health check is active**
```bash
docker compose logs caddy --tail=20
```
Caddy logs should show health check probes to app replicas. Alternatively:
```bash
curl -s http://localhost:8080/health
```
Expected: `{"status":"ok"}` — Caddy passes through to any healthy replica.

Collect all output. Display it in full before the checkpoint task.
