  <action>
Add a `caddy` service to `docker-compose.yml`. The final file must be:

```yaml
services:
  postgres:
    image: postgres:17
    environment:
      POSTGRES_USER: admin
      POSTGRES_PASSWORD: pass
      POSTGRES_DB: postgres
    volumes:
      - postgres-data:/var/lib/postgresql
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U admin -d postgres"]
      interval: 5s
      timeout: 5s
      retries: 5
      start_period: 10s

  app:
    build: .
    environment:
      DATABASE_URL: postgres://admin:pass@postgres:5432/postgres
      PGBOSS_MAX_CONNECTIONS: "5"
    depends_on:
      postgres:
        condition: service_healthy
    deploy:
      replicas: 6

  caddy:
    image: caddy:2
    ports:
      - "8080:8080"
    volumes:
      - ./Caddyfile:/etc/caddy/Caddyfile
    depends_on:
      - app

volumes:
  postgres-data:
    driver: local
```

Key points:
- Use official `caddy:2` image (latest stable Caddy v2)
- Port mapping `8080:8080` — per CADDY-03
- Mount `./Caddyfile:/etc/caddy/Caddyfile` — Caddy's default config path inside the container
- `depends_on: - app` — ensures app replicas start before Caddy begins routing (soft dependency; health checks validate liveness)
- Do NOT add ports to the `app` service — it must remain port-less (per COMP-04, already satisfied)
- Do NOT modify the `postgres` or `app` services
